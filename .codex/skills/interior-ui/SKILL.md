# Interior UI

Use this skill for interior-design websites, apps, landing pages, galleries, and client-facing tools where the visual system should feel polished, spatial, premium, and practical.

## Workflow

1. Identify the primary user journey: exploring projects, booking a consultation, comparing services, saving inspiration, or reviewing a proposal.
2. Make the first viewport show the actual interior-design subject through strong imagery, project visuals, material textures, or a useful interactive surface.
3. Prefer elegant, editorial layouts with clear hierarchy, generous image treatment, and fast scanning for room type, style, budget tier, timeline, and location.
4. Build real controls for expected workflows: filters, tabs, image galleries, before/after toggles, inquiry forms, save/share actions, and project detail views.
5. Verify responsive layouts so image crops, text blocks, buttons, and cards stay readable on mobile and desktop.

## Visual Direction

- Favor warm neutrals balanced with charcoal, muted green, stone, brass, terracotta, or deep accent colors.
- Avoid a flat beige-only look; include contrast through photography, material swatches, dark text, and crisp spacing.
- Use real or generated bitmap imagery for interiors whenever possible. Do not rely on abstract gradients or decorative blobs.
- Treat project cards as visual objects: image first, then project name, room type, location, style, and a concise action.
- Use swatches for materials and palettes, icon buttons for utility actions, segmented controls for room/style modes, and sliders for budget or area inputs.

## Content Patterns

- Keep copy concrete: room type, style, materials, timeline, scope, and consultation outcome.
- Replace generic labels like "Beautiful interiors" with specific phrases such as "Modular kitchen in walnut and stone" or "3BHK turnkey renovation".
- For forms, ask only for details needed to qualify the consultation: name, phone/email, city, property type, room count, budget range, and timeline.
